## Context

Photo IDs were migrated from integers to UUIDs using a zero-padding scheme (`00000000-0000-0000-0000-` + left-padded integer). These pass PostgreSQL's UUID type but fail the `uuid` npm library's `validate()` which requires RFC 4122 version/variant bits. All other UUID parameters in the system (`uuid`, `waveUuid`, `friendshipUuid`, `chatUuid`) are generated by `uuidv4()` and pass strict validation. The fix should be scoped to `photoId` only.

## Goals / Non-Goals

**Goals:**
- Fix `removePhotoFromWave` and `addPhotoToWave` runtime errors for migrated photo IDs
- Create a reusable utility for photoId validation that accepts legacy format
- Keep strict RFC 4122 validation for all other UUID parameters

**Non-Goals:**
- Changing validation for non-photoId UUIDs
- Fixing the legacy UUID data in the database
- Changing how new photo IDs are generated (they already use `uuidv4()`)

## Decisions

### 1. Scoped to photoId only
**Decision:** Only `photoId` parameters use relaxed validation. All other UUIDs keep `uuidValidate` from the `uuid` library.

**Rationale:** Only photo IDs were migrated from integers. User UUIDs, wave UUIDs, friendship UUIDs, chat UUIDs — all generated by `uuidv4()` — are valid RFC 4122 and should be validated strictly to catch malformed input.

### 2. Regex format check for photoId
**Decision:** Use `/^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i` to validate photoId shape.

**Rationale:** Accepts both legacy padded IDs (`00000000-0000-0000-0000-000000021212`) and new v4 UUIDs. Still rejects garbage input. Matches what PostgreSQL accepts.

### 3. Utility named `isValidPhotoId`
**Decision:** Place at `lambda-fns/utilities/isValidPhotoId.ts` with a single exported function.

**Rationale:** Name makes the intent clear — this is specifically for photo IDs that may be legacy format. Prevents accidental use for other UUID parameters where strict validation is desired.

## Risks / Trade-offs

- **[Broad scope]** 11 controller files change, but each change is small and mechanical (add import + guard clause). The 9 newly-validated controllers gain input safety they previously lacked.
- **[Naming clarity]** `isValidPhotoId` clearly signals this shouldn't be used for other UUIDs. All controllers that accept photoId now use a consistent validation pattern.
